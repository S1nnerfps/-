## 二、详细演示脚本

### 【0:00-0:30】开场

> 💬 "老师好，今天演示的项目叫 **Personal CS Coach** —— CS2 AI 驱动的实时战术助手。
> 
> 打 CS2 的都知道，死亡视角有 8 秒空闲时间。业余战队缺少战术教练，对手信息都浪费了。
> Personal CS Coach 就是让选手在这 8 秒里，在第二屏幕上随手画几笔，AI 就能分析出对手战术倾向。"

**操作：** 展示首页 7 张地图卡片，鼠标划过 Mirage hover 效果。

---

### 【0:30-2:00】编辑模式 — 预编辑战术

> 💬 "首先看战术编辑器。赛前你可以预先画好战术，AI 帮你评估优化。"

**操作步骤：**
1. 点击 Mirage 卡片 → 弹窗选"📐 编辑"
2. **选箭头** → 从 T 方出生点拖到 A 点（画一条红色进攻线）
3. 切换 **CT** 方 → **选墙壁** → 在 A 点前方画防守线（蓝色）
4. **选烟雾弹** → 在 A 点入口拖一个圈
5. **选燃烧弹** → 在 A 小道位置拖一个圈
6. 输入战术名"**爆A快攻**"，选"激进"风格
7. 点击 **🤖 AI 评估** → 展示五维雷达图 + 综合评分
8. 点击 **✨ AI 优化** → 展示优化前后对比
9. 点击 **💾 保存战术**

> 💬 "AI 给了 78 分，还提示控图覆盖率不足。优化后补充了防线和投掷物，评分提到 85。"

---

### 【2:00-3:30】比赛模式 — 实时标注

> 💬 "现在进入比赛。假设我们在打 Inferno，我是 CT 方刚阵亡。"
> 
> "阵亡后我看到 T 方进攻方向，用第二屏幕快速标记。"

**操作步骤：**
1. 首页 → 点击 Inferno → 弹窗选"🎮 比赛"
2. 保持 **CT** 方 → **选箭头** → 从 B 点拖到中路方向（标注对手进攻路线）
3. **选圆圈** → 在香蕉道位置圈出对手集结区
4. 切换 **T** 方 → **选箭头** → 从匪家拖到 A 点（标注对手主攻方向）
5. **选闪光弹** → 在 A 入口点一下（标注对手闪光弹爆点）
6. **选墙壁** → 在 A 拱门画防线
7. 再画两条箭头 (T 方 → A 点、T 方 → 拱门)

> 💬 "操作很简单：选工具 → 拖拽就画好了。CT 是蓝色、T 是黄色，颜色区分一目了然。"

---

### 【3:30-4:30】快速分析

> 💬 "回合间隙大概 15 秒，点击快速分析。"

**操作步骤：**
1. 点击 **⚡ 快速分析**
2. 展示分析结果：
   - **对手倾向**：进攻倾向右半区（A 点方向）
   - **置信度**：65%
   - **建议**："对手偏向进攻，可设防守陷阱"

> 💬 "3 秒内给出分析。注意置信度只有 65%——数据量还不够，这很合理。随着回合推进，置信度会提升。"

---

### 【4:30-5:30】深度分析 + AI 建议采纳

> 💬 "如果我们叫了暂停，就能做深度分析。"

**操作步骤：**
1. 点击 **⏸ 暂停分析** → 打开深度分析面板
2. 展示"对手画像"标签页：柱状图（主攻方向、防守站位分布）
3. 切换到"战术建议"标签页：
   - 展示 3 条战术建议
   - 每条旁边有 **➕ 应用标记** 按钮
4. 逐条点击"应用标记"→ 可以看到画板上实时添加了 AI 建议的标记
5. 最后点击 **✅ 采纳全部** → 一次性应用所有 AI 建议

> 💬 "AI 不仅给文字建议，还直接给出具体标记坐标。点击应用，标记就直接画到地图上。演示时这是很大的加分项。"

---

### 【5:30-6:30】商业 + 技术

> 💬 "技术上，原生 HTML + Canvas 纯前端，无需后端。API 可接入 OpenAI / DeepSeek / Claude 三种模型。"

**操作步骤：**
1. 返回首页 → 点击 **⚙ API 设置**
2. 展示提供商下拉（OpenAI / DeepSeek / Claude / 自定义）
3. 说明："已保存的 API Key 存在浏览器本地，不上传任何服务器。"
4. 返回首页 → 点击 **📊 车队数据中心**
5. 展示胜率、对局数、已保存战术数

> 💬 "商业模式上，我们采用 Freemium。免费版给单排玩家基础分析，付费版 $5/月给业余车队提供车队数据积累和深度分析。竞品如 CS2 Stratbook 是静态画板，没有 AI。"

---

## 三、Q&A 预演题库

### 技术方向

| 问题 | 回答要点 |
|------|----------|
| "AI 分析是本地还是云端？" | 双重：本地规则引擎提供即时分析（3-5s），接入 API（OpenAI/DeepSeek）提供深度分析。API 不可用时自动回退规则引擎。 |
| "怎么保证不违反 CS2 反作弊？" | 不读内存、不注入进程、不操作游戏客户端。纯靠玩家手动输入+第二屏幕显示，等同纸质战术本。 |
| "置信度怎么算的？" | 规则引擎：基于标记数量线性映射（每标记 +6%，上限 85%）。API 模式：由 LLM 自主评估。 |

### 商业方向

| 问题 | 回答要点 |
|------|----------|
| "谁付钱？为什么付？" | 业余战队 IGL（队内指挥）。他们需要数据分析但没有教练，$5/月/人，一个 5 人车队月费 $25，比请教练便宜 100 倍。 |
| "竞品怎么应对？" | Stratbook 是静态画板无 AI；YouTube 视频无法个性化适配用户打法。Personal CS Coach 的 AI 分析 + 多对局车队数据积累是差异化壁垒。 |
| "市场多大？" | CS2 日活约 80 万，业余战队估计 5-8 万支，目标渗透率 5% = 2500-4000 车队。 |

### 产品方向

| 问题 | 回答要点 |
|------|----------|
| "这个项目 AI 做了多少？" | AI 是效率工具。架构设计、交互逻辑、商业模式由我主导；AI 辅助编写 CSS 和 Canvas 代码框架，但核心算法（7 张地图数据结构、5 维评估逻辑、API 回退机制、标记拖动系统）我都能逐行解释。 |
| "为什么不用更复杂的 AI？" | 演示阶段规则引擎足够闭环。已预留 API 接口，可随时接入 GPT-4o 或 DeepSeek 提升分析深度。 |
| "数据安全？" | 全前端，数据存 localStorage，不上传服务器。API Key 同样本地存储。 |
| "后续规划？" | 语音播报分析结果、多地图切换、真实比赛 API 接入（Demo 解析）。 |
